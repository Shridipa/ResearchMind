## Key Design Decisions

### Retrieval-First Architecture
Answers are grounded in retrieved evidence. If no relevant sections found, the system returns "I couldn't find information about that" instead of generating plausible-sounding but potentially false answers.

### Multi-Model Summarization
Three transformer models with automatic fallback:
1. distilbart-cnn-12-6 (fast, general-purpose)
2. pegasus-arxiv (optimized for academic papers)
3. t5-small (lightweight)

If all fail, extractive summarization (always available) extracts key sentences based on keyword matching.

### Hybrid Retrieval Scoring
Combines semantic similarity (dense vectors) with keyword matching (BM25):
- Semantic: 65% weight → captures meaning and context
- Keywords: 35% weight → ensures exact terms found
- Result: Better recall and precision than either alone

### Lazy-Loading Transformers
Models load on first summarization request, not at startup. If a model fails to load, the system immediately tries the next fallback rather than crashing.

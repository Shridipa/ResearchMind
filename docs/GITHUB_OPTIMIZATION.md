# GitHub Portfolio Checklist

Use this checklist before publishing ResearchMind AI.

- Add screenshots to `docs/screenshots/`.
- Add a short demo GIF of upload, chat, and citations.
- Run backend tests and include the output in a release note.
- Run benchmark scripts on at least one public paper dataset.
- Add real benchmark values to the README table.
- Add architecture explanation to the repository description.
- Pin major dependency versions after the first working release.
- Add issues for reranking, streaming, experiment tracking, and persistent storage.
- Add a project board with `MVP`, `Research`, `Evaluation`, and `Deployment` lanes.

## Recruiter-Facing Highlights

- Evaluation-first RAG design
- Citation-grounded answering
- Modular ML pipeline abstractions
- FAISS vector search implementation
- Benchmarking harnesses and reproducible experiments
- Full-stack typed UI with FastAPI backend
